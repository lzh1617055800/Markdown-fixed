## 一、ChatBubble — 对话气泡

### 是什么

聊天面板上每句话就是一个气泡。用户说话→蓝色气泡靠右，AI 说话→灰色气泡靠左。

一个气泡有三种状态：

| 状态 | 何时 | 画面 |
|------|------|------|
| `Waiting` | AI 还没开始回复 | `...` 三个点循环闪烁（0.4s 一个节拍） |
| `Streaming` | AI 正在一个字一个字回复 | 文本逐渐变长 |
| `Done` | 回复完成 | 静态显示全文 |

### chatbubble.h — 完整代码

创建新文件 `chatbubble.h`，写以下内容：

```cpp
#ifndef CHATBUBBLE_H
#define CHATBUBBLE_H

#include <QWidget>
#include <QLabel>
#include <QTimer>

class ChatBubble : public QWidget
{
    Q_OBJECT    // Qt 元对象系统必备宏
public:
    enum Role { User, Assistant };
    enum State { Waiting, Streaming, Done };

    explicit ChatBubble(Role role, QWidget *parent = nullptr);

    void setText(const QString &text);        // 一次性设文本
    void appendText(const QString &chunk);    // 流式追加文本
    void setState(State state);               // 切换状态

    State state() const { return m_state; }
    Role  role()  const { return m_role; }
    QString text() const { return m_fullText; }

private:
    void updateStyle();   // 根据角色+状态刷新配色和圆角

    Role    m_role;
    State   m_state;
    QLabel *m_label;

    // 等待动画（三点闪烁）
    QTimer *m_waitingTimer;
    int     m_waitingDots;

    QString m_fullText;   // 流式过程中累积的完整文本
};

#endif // CHATBUBBLE_H
```

**逐行解释：**

- **`Q_OBJECT`**：这个宏必须放在类定义最上面（紧接 `{` 之后）。它告诉 Qt 的 moc（元对象编译器）为这个类生成信号/槽所需的运行时信息。不写的话 `connect()` 找不到对象，编译时报 `undefined reference to vtable`。

- **`enum Role { User, Assistant }`**：为什么不用 `bool isUser`？可读性——代码里 `createBubble(User)` 比 `createBubble(true)` 清楚十倍。而且将来要加 `System` 角色，enum 加一项就够。

- **`enum State { Waiting, Streaming, Done }`**：气泡生命周期。Waiting 是"请求发出去了但还没收到第一个字"，Streaming 是"已经开始收到字并显示"，Done 是"这条消息已结束"。

- **`setText` vs `appendText`**：用户气泡一次性设文本（`setText`），AI 气泡分多次追加（`appendText`）。appendText 内部做的事情是 `m_fullText += chunk; m_label->setText(m_fullText);` ——每次追加后重设整个文本。

- **`m_waitingTimer`**：Waiting 状态下的 "..." 三点动画，每 400ms 循环变化点数：`"" → "." → ".." → "..." → ""`。

- **`m_fullText`**：流式追加过程中保存完整文本。如果不用这个变量，每次 append 时 QLabel 里的文本是不是完整？答：是完整的，但取不出来。存一份在 m_fullText 里，`responseDone()` 时就能把全文写入 ChatModel 存档。

- **`updateStyle()`**：切换角色/状态时重新设置 QLabel 的 StyleSheet（背景色、圆角、边框）。封装成独立函数避免代码重复。

### chatbubble.cpp — 完整代码

创建新文件 `chatbubble.cpp`：

```cpp
#include "chatbubble.h"
#include <QHBoxLayout>

ChatBubble::ChatBubble(Role role, QWidget *parent)
    : QWidget(parent)
    , m_role(role)
    , m_state(Done)         // 初始状态：完成
    , m_waitingDots(0)
{
    // ── 1. 文本标签 ──
    m_label = new QLabel(this);
    m_label->setWordWrap(true);              // 长文本自动换行
    m_label->setTextFormat(Qt::PlainText);   // 纯文本，防止 HTML 注入
    m_label->setMaximumWidth(500);           // 气泡最宽 500px

    // ── 2. 等待动画定时器 ──
    m_waitingTimer = new QTimer(this);
    connect(m_waitingTimer, &QTimer::timeout, this, [this]() {
        m_waitingDots = (m_waitingDots + 1) % 4;   // 0→1→2→3→0
        m_label->setText(QString(m_waitingDots, '.'));
    });

    // ── 3. 布局：addStretch() 弹簧决定气泡靠左还是靠右 ──
    QHBoxLayout *outerLayout = new QHBoxLayout(this);
    outerLayout->setContentsMargins(8, 4, 8, 4);

    if (m_role == User) {
        outerLayout->addStretch();       // 弹簧在左边→推气泡到右边
        outerLayout->addWidget(m_label);
    } else {
        outerLayout->addWidget(m_label);
        outerLayout->addStretch();       // 弹簧在右边→推气泡到左边
    }

    updateStyle();
}

void ChatBubble::setText(const QString &text)
{
    m_fullText = text;
    m_label->setText(text);
}

void ChatBubble::appendText(const QString &chunk)
{
    m_fullText += chunk;
    m_label->setText(m_fullText);
}

void ChatBubble::setState(State state)
{
    m_state = state;

    if (state == Waiting) {
        m_waitingTimer->start(400);    // 每 400ms 触发一次，更新点数
    } else {
        m_waitingTimer->stop();
        if (state == Done && m_fullText.isEmpty()) {
            m_label->setText(QStringLiteral("(empty response)"));
        }
    }

    updateStyle();
}

void ChatBubble::updateStyle()
{
    QString bg, fg, border;

    if (m_role == User) {
        bg     = QStringLiteral("#0078D4");   // 蓝色底
        fg     = QStringLiteral("#FFFFFF");   // 白色字
        border = QStringLiteral("#0078D4");
    } else {
        bg     = m_state == Waiting ? QStringLiteral("#F0F0F0")
                                    : QStringLiteral("#F5F5F5");
        fg     = QStringLiteral("#1A1A1A");   // 深灰字
        border = QStringLiteral("#E0E0E0");
    }

    m_label->setStyleSheet(QString(
        "QLabel {"
        "  background-color: %1;"
        "  color:            %2;"
        "  border:           1px solid %3;"
        "  border-radius:    12px;"
        "  padding:          10px 14px;"
        "  font-size:        13px;"
        "}"
    ).arg(bg, fg, border));
}
```

**逐行解释：**

- **布局技巧 `addStretch()`**：QHBoxLayout 里的 `addStretch()` 就像一个弹簧，吃掉所有剩余空间。弹簧在 label 左边→label 被推到最右（用户气泡）。弹簧在 label 右边→label 被推到最左（AI 气泡）。

- **`setTextFormat(Qt::PlainText)`**：QLabel 默认是 `Qt::AutoText`——如果文本长得像 HTML（比如包含 `<script>`），Qt 会把它当 HTML 渲染。设成 PlainText 强制纯文本，安全。

- **`setMaximumWidth(500)`**：气泡最长 500 像素。不设这个的话，用户灌进去一段 5000 字的文章，气泡会撑满整个屏幕宽度。

- **`QString(m_waitingDots, '.')`**：Qt 的 QString 构造函数，`QString(3, '.')` 得到 `"..."`。定时器每次触发，dots 从 0 变到 3 再归零，效果就是 `. → .. → ... →  → . → ...`。

- **`m_waitingTimer->start(400)`**：400 毫秒间隔——太短（100ms）闪得太快刺眼，太长（1000ms）显得反应慢。400ms 是一个常见的加载动画节奏。

- **`QStringLiteral`**：C++ 的 `QString("hello")` 在运行时从 UTF-8 转成 UTF-16。`QStringLiteral("hello")` 在编译期完成转换，存在只读数据段。不常变的字符串用它更高效。短的字符串（20 字符以下）用不用差别不大，但养成习惯是好的。

- **`setStyleSheet` 里 `.arg(bg, fg, border)`**：`%1` 被 bg 替换，`%2` 被 fg 替换，`%3` 被 border 替换。`border-radius: 12px` 是圆角半径。

## 二、ChatModel — 对话历史

### 设计原则

ChatModel **不继承 QWidget**，不包含任何 UI 代码。它是纯数据层。

好处：
- UI 层崩了，对话历史不丢
- 将来写单元测试，不需要开窗口
- 存盘/读盘只改 ChatModel，不碰 UI

### chatmodel.h — 完整代码

```cpp
#ifndef CHATMODEL_H
#define CHATMODEL_H

#include <QString>
#include <QDateTime>
#include <QVector>
#include <QJsonArray>
#include <QJsonObject>

class ChatModel
{
public:
    enum Role { User, Assistant, System };

    struct Message {
        Role      role;
        QString   content;
        QDateTime timestamp;
    };

    void addMessage(Role role, const QString &content);
    const QVector<Message>& messages() const { return m_messages; }
    void clear();

    // 转为 OpenAI 兼容的 messages 数组
    QJsonArray toJsonArray() const;

private:
    QVector<Message> m_messages;
    static QString roleToString(Role role);
};

#endif // CHATMODEL_H
```

### chatmodel.cpp — 完整代码

```cpp
#include "chatmodel.h"

void ChatModel::addMessage(Role role, const QString &content)
{
    m_messages.append({role, content, QDateTime::currentDateTime()});
}

void ChatModel::clear()
{
    m_messages.clear();
}

QJsonArray ChatModel::toJsonArray() const
{
    QJsonArray arr;
    for (const auto &msg : m_messages) {
        QJsonObject obj;
        obj["role"]    = roleToString(msg.role);
        obj["content"] = msg.content;
        arr.append(obj);
    }
    return arr;
}

QString ChatModel::roleToString(Role role)
{
    switch (role) {
    case User:      return QStringLiteral("user");
    case Assistant: return QStringLiteral("assistant");
    case System:    return QStringLiteral("system");
    }
    return QStringLiteral("user");
}
```

**逐行解释：**

- **`Role::System`**：三个角色——User（用户说的话）、Assistant（AI 回复）、System（给 AI 的角色设定，不显示在气泡里）。DeepSeek API 要求 messages 数组里每条都有 role 字段，值必须是 `"user"` / `"assistant"` / `"system"` 之一。

- **`toJsonArray()`**：把内存里的消息列表转成 DeepSeek/OpenAI API 需要的格式：
  ```json
  [
    {"role": "system",    "content": "你是写作助手"},
    {"role": "user",      "content": "帮我润色这段"},
    {"role": "assistant", "content": "好的，润色后..."},
    {"role": "user",      "content": "再修改一下语气"}
  ]
  ```
  这就是"多轮对话"的本质——每次请求都把之前所有对话历史带上，AI 才"记得"前面说过什么。

- **不继承 QObject**：ChatModel 不需要信号/槽，它只是一个数据容器。把它做成普通的 C++ 类（值对象）更轻量。

## 三、AiClient — 抽象接口

### 为什么需要这个

将来 DeepSeekClient 和 ChatPanel 通过 AiClient 接口交互。如果以后要换成 GPT、文心一言，只写一个新的 GptClient 继承 AiClient 就行，ChatPanel 一行不改。

这个设计模式叫 **依赖倒置（Dependency Inversion）**——高层模块（ChatPanel）不依赖低层模块（DeepSeekClient），两者都依赖抽象（AiClient）。

### aiclient.h — 完整代码

**只需要头文件，没有 .cpp！** 因为全是纯虚函数。

```cpp
#ifndef AICLIENT_H
#define AICLIENT_H

#include <QObject>
#include <QJsonArray>

class AiClient : public QObject
{
    Q_OBJECT
public:
    explicit AiClient(QObject *parent = nullptr) : QObject(parent) {}
    virtual ~AiClient() = default;

    // 子类必须实现这两个
    virtual void sendMessage(const QJsonArray &messages) = 0;
    virtual void cancel() = 0;

signals:
    void streamChunk(const QString &chunk);    // 收到一个文本片段
    void responseDone();                       // 本次响应结束
    void errorOccurred(const QString &error);  // 出错
};

#endif // AICLIENT_H
```

**逐行解释：**

- **`virtual ... = 0`**：C++ 纯虚函数语法。`= 0` 意思是"这个函数没有默认实现，子类必须 override"。包含纯虚函数的类叫"抽象类"，不能直接实例化（`new AiClient()` 编译报错）。

- **`virtual ~AiClient() = default`**：基类析构函数必须是虚函数。否则 `AiClient *p = new DeepSeekClient(); delete p;` 只会调 AiClient 的析构函数，DeepSeekClient 的资源泄漏。

- **三个信号的设计逻辑**：
  - `streamChunk(chunk)`——每个文本片段触发一次，ChatPanel 收到后追加到当前 AI 气泡
  - `responseDone()`——流结束时触发一次，ChatPanel 标记气泡 Done + 存历史 + 解锁输入
  - `errorOccurred(err)`——网络/API 错误，ChatPanel 显示错误信息 + 解锁输入

- **为什么用信号而不用回调函数？** 信号/槽是 Qt 的核心机制，线程安全（同线程直接调用，跨线程自动排队）。最重要的是 ChatPanel 只需要 `#include "aiclient.h"`，不需要 include DeepSeekClient 的头文件——编译依赖降到最低。
