三种 AI 面板布局方案

参考 VS Code 中 Claude Code 的交互模式，AI 助手需要一个对话区域

A

底部面板（推荐）

编辑区下方新增可折叠的 AI 对话面板，类似 VS Code Terminal / Claude Code 面板

布局：行号 | 编辑区 | 预览区（上方） + AI 对话面板（下方）
1
2
3
编辑区
预览区
🤖 AI 助手 可折叠 ▾
AI: 要我帮你优化这段 Markdown 吗？
▸ 输入你的问题...

Pros

  • 不动现有 3 栏布局
  • 可折叠，不干扰写作
  • 最像 VS Code Claude Code 体验
  • 宽幅显示，适合长对话

Cons

  • 纵向空间被压缩
  • 小屏幕体验不佳
B

右侧替换预览区（标签切换）

预览区和 AI 对话区共用右侧面板，通过标签切换

布局：行号 | 编辑区 | 预览 / AI（标签切换）

Pros

  • 不压缩纵向空间
  • 空间利用率高

Cons

  • 预览和 AI 不能同时看
  • 频繁切换体验差
  • 和"实时预览"核心功能冲突
C

独立浮动窗口

AI 助手作为独立的 QDialog/QDockWidget，可拖拽到任意位置

布局：主窗口不变 + AI 独立浮动窗口

🤖 AI 助手 ▸

输入消息...

Pros

  • 最灵活，不挤占主窗口
  • 多显示器友好

Cons

  • 需要额外管理窗口生命周期
  • 和主编辑器"一体化"体验弱
  • 小屏幕下窗口叠窗口

推荐：方案 A（底部面板）

理由：

  • VS Code Claude Code 就是底部面板，用户心智一致
  • 不破坏现有的 3 栏编辑体验
  • 实现难度适中，用 QSplitter 纵向拆分即可
  • 可折叠设计让用户按需打开
