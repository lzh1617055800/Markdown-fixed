# LightMark 深化方向清单

> 按优先级分三档：现在就值得做的 → 有时间的进阶 → 长期愿景

## 一、AI 接口集成（最值得做的下一步）

### 1.1 整体思路

在编辑区旁边加一个 AI 对话面板。选中 Markdown 文本 → 右键菜单 → 选择 AI 操作 → 调用 API → 结果插回编辑区。用 OpenAI 兼容接口（任何一家都支持这个格式：OpenAI、DeepSeek、通义千问、智谱 GLM）。

### 1.2 架构设计

```
┌──────────┬──────────────┬──────────────┐
│ 行号区域  │  编辑区       │  AI 面板      │
│          │              │  (新增)       │
│          │              │  ┌──────────┐ │
│          │              │  │ 对话历史  │ │
│          │              │  │          │ │
│          │              │  │ AI回答   │ │
│          │              │  ├──────────┤ │
│          │              │  │ 输入框    │ │
│          │              │  │ [发送]   │ │
│          │              │  └──────────┘ │
└──────────┴──────────────┴──────────────┘
```

### 1.3 分步实现

**第 1 步：新增 AIClient 类**

```cpp
// aiclient.h — 封装 OpenAI 兼容 API 调用
class AIClient : public QObject {
    Q_OBJECT
public:
    void setApiKey(const QString &key);
    void setBaseUrl(const QString &url);  // 默认 https://api.openai.com/v1
    void setModel(const QString &model);  // 默认 gpt-4o
    void sendMessage(const QString &userMessage);

signals:
    void responseReady(const QString &response);
    void errorOccurred(const QString &error);

private:
    QNetworkAccessManager *m_networkManager;
    QString m_apiKey;
    QString m_baseUrl;
    QString m_model;

    // 把对话历史 + 新消息拼成 OpenAI chat completions 请求体
    QJsonObject buildRequestBody(const QString &userMessage);
};
```

**第 2 步：新增 AI 面板（AIPanel 类）**

右侧新增一个可折叠面板，包含：
- 对话历史区（QTextEdit 只读）
- 输入框 + 发送按钮
- 快捷操作按钮：续写 / 润色 / 翻译成英文 / 总结

**第 3 步：新增设置对话框（AISettingsDialog）**

让用户配置：
- API 地址（默认 https://api.openai.com/v1，可以换成 DeepSeek/通义千问的地址）
- API Key
- 模型名称
- 保存到 QSettings，下次启动记住

**第 4 步：右键菜单集成**

编辑器右键菜单新增 AI 选项：
- "AI 续写" → 取光标前 500 字作为上下文 → 发送给 AI → 返回续写内容插到光标位置
- "AI 润色" → 选中文本 → 发送"请润色以下文字" → 替换选中文本
- "AI 翻译为英文" → 选中文本 → 翻译 → 替换
- "AI 总结" → 全文 → 总结 → 插入文档末尾

### 1.4 为什么用 OpenAI 兼容接口

所有主流 AI 服务商都兼容 OpenAI 的 chat/completions 格式。你写好一套代码，用户填哪个服务的地址和 Key 就能用哪个：

| 服务商 | baseUrl | 模型示例 |
|--------|---------|---------|
| OpenAI | https://api.openai.com/v1 | gpt-4o |
| DeepSeek | https://api.deepseek.com | deepseek-chat |
| 通义千问 | https://dashscope.aliyuncs.com/compatible-mode/v1 | qwen-plus |
| 智谱 GLM | https://open.bigmodel.cn/api/paas/v4 | glm-4 |
| 本地 Ollama | http://localhost:11434/v1 | llama3 |

### 1.5 请求体格式（一次讲清楚）

发一个 POST 到 `{baseUrl}/chat/completions`：

```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "system", "content": "你是一个Markdown写作助手"},
    {"role": "user", "content": "帮我续写这段文字：..."}
  ],
  "temperature": 0.7,
  "stream": false
}
```

HTTP Header 里加 `Authorization: Bearer {apiKey}`。

### 1.6 新增文件预估

```
aisettingsdialog.h/cpp    ~200 行  配置对话框
aiclient.h/cpp            ~250 行  API 调用封装
aipanel.h/cpp             ~300 行  AI 对话面板 UI
```

不需要引入任何第三方库——Qt 自带的 `QNetworkAccessManager` 就能发 HTTP 请求，`QJsonDocument` 处理 JSON。

## 二、现在就值得做的优化

### 2.1 Markdown 解析升级为 AST

**现状**：正则流水线，按优先级顺序对同一根字符串反复替换。嵌套语法（列表里的代码块）处理困难。

**目标**：先词法分析把文本切成 Token 流（Heading、Bold、Code、Text 等），再语法分析构建树。

**为什么值得做**：面试含金量最高——"我用编译原理的前端技术重构了 Markdown 解析器"比"我用正则替换"强一个数量级。

### 2.2 真正的视图级差分渲染

**现状**：差分渲染只保持滚动条位置，`setHtml()` 仍然全量重建 QTextDocument。

**目标**：用 `QTextCursor` 在 QTextDocument 上做 Block 级别的局部替换。只删改变化行对应的 block，不改动其他 block。

**效果**：大文档预览更新从 O(n) 降到 O(变化行数)。

### 2.3 虚拟化编辑器光标精确追踪

**现状**：找光标位置用 `m_visibleStartLine * 80` 近似估算全局位置，80 是按每行 80 字符猜的。

**目标**：维护一个 `m_lineCharCounts: QVector<int>` 记录每行字符数，光标全局位置 = sum(m_lineCharCounts[0..line-1]) + column。O(1) 查询，编辑时增量更新。

### 2.4 多标签页支持

**现状**：只能打开一个文件，新建会关闭当前文件。

**目标**：用 QTabWidget 管理多个编辑实例。每个标签页有独立的 VirtualizedEditor + 文件路径 + 修改状态。关闭标签时检查未保存内容。

### 2.5 最近文件列表

**现状**：`AppSettings` 已实现了 `saveRecentFiles()` 和 `loadRecentFiles()`，但 UI 里没接入。

**目标**：文件菜单底部显示最近 5 个文件，点击直接打开。启动时加载最近文件列表。这个改动不到 50 行。

## 三、有时间的进阶优化

### 3.1 扩展 Markdown 语法

| 语法 | 难度 | 说明 |
|------|------|------|
| 脚注 `[^1]` | 低 | 底部自动收集脚注内容 |
| 任务列表 `- [ ]` `- [x]` | 低 | 渲染为可勾选的 checkbox |
| 删除线 `~~text~~` | 低 | 正则 `~~(.*?)~~` → `<del>` |
| Mermaid 图表 | 高 | 集成 mermaid.js 渲染流程图/时序图 |
| 数学公式 `$x^2$` `$$...$$` | 中 | 集成 KaTeX/MathJax |
| 高亮 `==text==` | 低 | 正则 → `<mark>` |
| 目录 `[TOC]` | 中 | 扫描标题自动生成目录链接 |

### 3.2 编辑器体验增强

| 功能 | 说明 |
|------|------|
| 快捷键补全 | 输入 `**` 自动补全 `**`，光标放在中间 |
| 括号匹配高亮 | 光标在 `(` 上时高亮配对的 `)` |
| 拖拽移动行 | 选中一行用 Alt+↑↓ 上下移动 |
| 多光标编辑 | Ctrl+D 选中下一个相同词 |
| 自动补全路径 | 输入 `![` 时弹出文件选择器 |

### 3.3 UI 改进

| 功能 | 说明 |
|------|------|
| 设置对话框 | 统一管理字体、字号、自动保存间隔、默认编码 |
| 可拖拽的预览区位置 | 预览区可以在右侧/底部切换 |
| 专注模式 | 全屏 + 居中窄栏 + 隐藏工具栏 |
| 打字机模式 | 当前编辑行始终保持在屏幕中央 |
| 大纲视图 | 左侧显示标题树，点击跳转 |

### 3.4 工程化

| 事项 | 说明 |
|------|------|
| 单元测试 | GTest 覆盖 MarkdownParser 每种语法的输入→预期输出 |
| CI/CD | GitHub Actions 自动编译 Windows + Linux |
| 打包发布 | CPack 或 windeployqt 生成安装包 |
| 国际化 | Qt Linguist 做中英文界面切换 |
| Doxygen 文档 | 注释规范 → 自动生成 API 文档 |

## 四、长期愿景

- **插件系统**：Lua/Python 脚本扩展，用户写插件加新语法
- **协作编辑**：WebSocket 多人实时协作（CRDT 算法）
- **移动端**：用 Qt for Android/iOS 迁移到手机
- **VSCode 插件版**：把解析器抽出来做成 VSCode 扩展的渲染引擎
- **Git 集成**：编辑器内显示 diff、stage/unstage、commit

## 五、建议的实施路线

```
第 1 个月：AI 接口（AIClient + AIPanel + 设置对话框）
第 2 个月：最近文件列表 + 多标签页 + 设置对话框
第 3 个月：Markdown AST 重构
第 4 个月：视图级差分渲染 + 光标精确追踪
第 5-6 个月：扩展语法 + 编辑器体验增强
```

AI 接口排第一因为：它的代码路径完全独立（新增 3 个文件），不影响现有逻辑；面试能说"我接入了大模型 API"；Qt 的 QNetworkAccessManager 写 HTTP 请求是很好的学习点。
